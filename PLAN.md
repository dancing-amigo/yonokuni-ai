# 学習開始までのロードマップ

Yonokuni AI はルール実装・Gym 環境・MCTS・自己対戦／学習ループの最小構成が整っている。学習本番に移る前に、以下の機能を優先的に実装する。

## 1. ロギングとチェックポイント
- [ ] TensorBoard/W&B などで `loss`, `value`, `policy_entropy`, `buffer_size`, `self_play_winrate` を可視化。
- [ ] モデル重み・オプティマイザ・リプレイバッファを定期保存し、再開用ロード関数を実装。
- [ ] 設定値（MCTS 回数、温度、バッチサイズなど）を Hydra/JSON で外部化し再現性を確保。

## 2. 評価およびゲーティング準備
- [ ] ルールベース Bot/ランダム/旧モデルと対戦させる評価ジョブを作成。
- [ ] 勝率・平均手数・中央勝利率・死に駒発生ターンなどを集計しダッシュボード化。
- [ ] ゲーティング（新旧モデル対戦で勝率基準を満たした場合のみ昇格）フローを実装。

## 3. コンソール対戦
- [ ] 人間 vs 学習済みモデルを CLI で対戦できるスクリプトを作成（合法手一覧表示、手番入力、終了判定、簡易 MCTS 推論）。
- [ ] モデル選択・探索回数・温度などを CLI 引数から指定可能にする。

## 4. 自己対戦スケーリングとデータ品質
- [ ] 自己対戦ワーカーをマルチプロセス化して探索並列数を拡張。
- [ ] 温度スケジュール（序盤 τ=1 → 終盤 τ→0）、探索回数プリセット（学習用/早指し）を設定化。
- [ ] 収集データの自動検証（NaN、勝率ドリフト、手数統計、色バランス）を定期実行。

## 5. 推論ユーティリティの強化
- [ ] Mixed precision・バッチ推論・TorchScript/ONNX エクスポートを整備し、評価や推論サービスで利用可能にする。
- [ ] モデルのバージョニング API とメタデータ（訓練日・勝率・設定）管理を実装。

以上を完了後、長時間自己対戦＋学習ジョブを投入し、定期的な評価とゲーティングで性能をモニタリングする。
